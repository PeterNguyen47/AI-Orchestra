# Repository Instructions

Before editing, read the relevant GitHub issue and the product, architecture, security, and delivery-governance documentation.

- Use one bounded issue, one branch, and one pull request.
- Keep at most one major item In Progress.
- Do not expand scope without a recorded scope decision.
- Never commit secrets, credentials, tokens, private keys, or session values.
- Keep all model calls server-side.
- Use TypeScript strict mode when implementation begins.
- Add tests and documentation with behavior changes.
- Do not modify unrelated files.
- Run repository-defined lint, typecheck, test, and build commands when they exist.
- Report changed files, commands, validation, risks, blockers, and the exact next action.

GitHub Project #2 is authoritative for execution status. Node claims must use `executable`, `simulated`, or `roadmap`; never imply simulated or planned behavior is executable.

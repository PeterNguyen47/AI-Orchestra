# Contributing

## Definition of Ready

Work has a bounded issue, reason, scope, acceptance criteria, non-goals, dependencies, validation plan, documentation impact, priority, and scope classification. Required decisions and inputs are available.

## Branch naming

Use `<type>/<issue-key>-<short-description>`, for example `feat/ao-002-app-foundation`. One issue maps to one branch and one pull request unless a documented scope decision says otherwise.

## Pull requests

Link the issue, summarize scope and changed files, provide validation evidence, complete the security review, add screenshots when relevant, and confirm unrelated work is excluded. Keep the change reviewable and do not merge with failing checks or unresolved review feedback.

## Definition of Done

Acceptance criteria are met; tests and documentation are current; security and secret checks pass; required lint, typecheck, test, and build commands pass; Project status and completion evidence are synchronized; and the PR is merged.

## Scope changes

Stop implementation, record the proposal and rationale in `docs/project-management/SCOPE_CHANGE_LOG.md`, update the issue and affected documentation, and obtain an explicit decision before continuing.

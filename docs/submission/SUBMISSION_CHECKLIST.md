# Submission Checklist

## Manual legal and account gates

- [ ] Entrant personally reads and acknowledges the current [Official Rules](https://openai.devpost.com/rules). This repository does not record acceptance on the entrant's behalf.
- [ ] Request available free Codex credits from the [Resources page](https://openai.devpost.com/resources) before July 17, 2026 at 3:00 PM Eastern; retain evidence before marking complete.

## Working submission

- [ ] Working, coherent project passes the clean judge path.
- [ ] Developer Tools category is selected.
- [ ] Project description explains problem, audience, features, implementation, and impact.
- [ ] Public YouTube demo is under three minutes and shows the project working.
- [ ] Video audio explains how Codex and GPT-5.6 were used.
- [ ] Repository URL is public and Apache-2.0 licensed.
- [ ] README contains tested setup, supported-platform, and run instructions.
- [ ] Sample data is included when required and contains no sensitive data.
- [ ] Codex usage evidence and key decisions are documented.
- [ ] GPT-5.6 usage evidence identifies the actual executable path.
- [ ] Judge test path works without rebuilding from scratch and includes seeded demo credentials safely.
- [ ] Primary Codex `/feedback` Session ID is recorded from the session where most core functionality was built.
- [ ] Architecture claims label executable, simulated, and roadmap nodes accurately.
- [ ] Final security, secret, license, accessibility, and regression checks pass.
- [ ] Devpost submission is finalized before July 21, 2026 at 8:00 PM Eastern.

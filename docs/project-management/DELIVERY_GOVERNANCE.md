# Delivery Governance

GitHub is authoritative for scope, architecture decisions, risks, execution status, and completion evidence. Project #2 is the execution board.

## Controls

- One bounded issue, branch, and pull request.
- At most one major issue In Progress.
- `Ready` means the issue satisfies the Definition of Ready and is the authorized next item.
- `Review` means implementation is complete and evidence is under review.
- `Blocked` requires a named impediment and owner.
- `Done` requires merged work and synchronized evidence.
- MVP expansion requires an approved entry in the scope-change log.
- No merge with failing checks, conflicts, unresolved required review, secret exposure, or unmet acceptance criteria.

## Evidence

Each PR records changed files, commands, results, risks, blockers, and next action. Claims distinguish planned, simulated, and executable behavior.
